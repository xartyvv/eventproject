# EventFinder — Сайт с умным поиском мероприятий

Проект выполнен командой из 2 человек. Технологии: GoLang для бэкенда, JavaScript для фронтенда, PostgreSQL для хранения данных.

Веб-сервис афиши мероприятий с системой умного поиска и ранжирования на основе **метода анализа иерархий (МАИ)**.

## Функциональность

- **Создание событий** — пользователи могут публиковать карточки мероприятий (название, описание, дата, место, 10 критериев)
- **Система ранжирования (МАИ)** — пользователь формирует предпочтения через матрицу парных сравнений (шкала 1-9), система вычисляет веса и ранжирует мероприятия
- **Базовые фильтры** — поиск по категориям и датам
- **Личный кабинет и Избранное** — регистрация, сохранение мероприятий в избранный список, сохранение приоритетов (матрицы сравнений)
- **JWT авторизация** — безопасная аутентификация
- **Поиск по названию и описанию** — умный текстовый поиск по афише
- **Категории и датировка** — фильтрация по категориям и диапазону дат
- **Сохранение профиля МАИ** — матрицы парных сравнений хранятся для пользователя

## 10 критериев для МАИ

| № | Критерий | Тип | Описание |
|---|----------|-----|----------|
| 1 | Стоимость билета | Числовой | В рублях (меньше = лучше) |
| 2 | Длительность | Числовой | В часах |
| 3 | Вместимость | Числовой | Кол-во человек |
| 4 | Выходной день | Булевый | true/false |
| 5 | Онлайн формат | Булевый | true/false |
| 6 | Возрастное ограничение | Числовой | 0 = без ограничений |
| 7 | Требуется регистрация | Булевый | true/false |
| 8 | Рейтинг организатора | Числовой | 0-10 |
| 9 | Время проведения | Числовой | 1=утро, 2=день, 3=вечер, 4=ночь |
| 10 | Интерактивность | Числовой | 0-10 |

## Установка и запуск

### Способ 1: Docker (рекомендуется)

Самый простой способ — использовать Docker Compose. Всё (PostgreSQL + бэкенд) запустится автоматически.

```bash
# Запуск приложения на Windows
start.bat

# Остановка
stop.bat
```

Для macOS / Linux:

```bash
# Запуск приложения
./start.sh

# Остановка
./stop.sh
```

Или вручную:

```bash
# 1. Запустить PostgreSQL
docker-compose up -d postgres

# 2. Подождать 10 секунд (пока БД инициализируется)
timeout /t 10

# 3. Запустить бэкенд
docker-compose up -d backend

# Просмотр логов
docker-compose logs -f backend

# Остановка (контейнеры будут просто остановлены, данные сохранятся)
docker-compose stop

# Повторный запуск контейнеров
docker-compose start

# Остановка с удалением контейнеров и данных БД
docker-compose down -v
```

Приложение будет доступно по адресу: **http://localhost:8080**

**Опционально: pgAdmin** (веб-интерфейс для управления БД):
```bash
docker-compose --profile tools up -d pgadmin
```
Откройте **http://localhost:5050**
- Email: `admin@admin.com`
- Пароль: `admin`

### Алгоритм МАИ (Метод анализа иерархий)

**Реализация:** Метод собственных векторов через геометрическое среднее

**Этапы:**
1. Построение матрицы парных сравнений 10×10 (шкала Саати от 1 до 9)
2. Вычисление геометрического среднего каждой строки
3. Нормализация весов (сумма = 1)
4. Min-Max нормализация критериев мероприятий (диапазон 0-1)
5. Инвертирование "плохих" критериев (стоимость, возрастное ограничение)
6. Вычисление итогового скор: `Score = Σ(weight_i * normalized_value_i)`
7. Сортировка по убыванию score

**Преимущества:**
- Прозрачность и объяснимость весов
- Математическая обоснованность (Thomas L. Saaty, 1980)
- Возможность проверки согласованности суждений (Consistency Ratio)

## Руководство пользователя

### 1. Регистрация и вход

1. Перейдите на страницу регистрации
2. Введите email, имя пользователя и пароль (минимум 6 символов)
3. После регистрации вы будете автоматически авторизованы

### 2. Создание мероприятия

1. Перейдите в раздел "Создать"
2. Заполните основную информацию (название, описание, дата, место)
3. Установите значения 10 критериев (стоимость, длительность, вместимость и др.)
4. Нажмите "Создать мероприятие"

### 3. Настройка ранжирования (МАИ)

1. Перейдите в раздел "Ранжирование"
2. Для каждой пары критериев укажите, насколько один важнее другого (шкала 1-9)
   - 1 = одинаково важны
   - 3 = умеренное превосходство
   - 5 = сильное превосходство
   - 7 = очень сильное превосходство
   - 9 = абсолютное превосходство
3. Нажмите "Сохранить и вычислить веса"
4. Система отобразит вычисленные веса и ранжированные мероприятия

### 4. Добавление в избранное

1. На странице мероприятий нажмите "В избранное" на карточке
2. Просмотреть избранные можно в разделе "Избранное"

### 5. Фильтрация мероприятий

1. На странице "Мероприятия" используйте фильтры
2. Выберите категорию и/или укажите диапазон дат
3. Нажмите "Применить"
